# Methodik

## Evidenzklassen
- **E1** - Primary Internal Observation
- **E2** - Primary External Source
- **E3** - Secondary Source
- **E4** - Inference
- **E5** - Design Decision
- **E6** - Open Hypothesis

## Kerndisziplin
Starke öffentliche Aussagen sollten nie ohne drei Dinge erscheinen:
1. Claim-Typ
2. Evidenzklasse
3. Quellenpfad

## Trennregeln
### Beobachtung
Aussage ueber etwas, das direkt im aktuellen System oder Artefaktbestand vorliegt.

### Ableitung
Interpretation, die aus mehreren Beobachtungen gezogen wird.

### Designentscheidung
Governance- oder Betriebsentscheidung dieses Projekts.

### Hypothese
Plausible, aber noch nicht belastbar etablierte Aussage.

## Claim-to-Source-Regel
Jede staerkere Aussage sollte auf einen Claim-Eintrag verweisen mit:
- Claim ID
- DE/EN-Formulierung
- Claim-Typ
- Evidenzklasse
- Support-Status
- Quellenliste
- Caveats

## Public-Writing-Regel
Nicht zulassen, dass:
- Ableitung wie Beobachtung klingt
- Governance wie Evidenz klingt
- Ambition wie aktueller Ist-Zustand klingt

## Messdisziplin
Ein kuenftiger Closed-Loop-Proof sollte offenlegen:
- was sich geaendert hat
- warum es sich geaendert hat
- wann es sich geaendert hat
- welche Wirkung erwartet wurde
- was spaeter beobachtet wurde
- welche alternativen Erklaerungen bleiben

Siehe `messgrenzen.md` fuer die derzeitigen Grenzen der oeffentlichen Evidenz und Observability.
