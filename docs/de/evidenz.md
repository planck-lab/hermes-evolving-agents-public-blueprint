# Evidenzregister

Diese Seite definiert die minimale öffentliche Claim-Disziplin für die Roadmap.

## Kernregel
Jede relevante Aussage sollte als eines von vier Dingen markiert werden:
- **Beobachtung**
- **Ableitung**
- **Designentscheidung**
- **Hypothese**

## Aktueller öffentlicher Claim-Satz
| Claim ID | Typ | Kurzform | Status |
|---|---|---|---|
| C-001 | Ableitung | Das System liegt aktuell ungefaehr bei Phase 1.5. | partially-supported |
| C-002 | Ableitung | Bridge Closure ist derzeit wahrscheinlich der Hauptengpass. | partially-supported |
| C-003 | Beobachtung | Messflaechen, Signal-Logik und Review-/Adoption-Staging existieren bereits. | supported |
| C-004 | Beobachtung | Ein stabiler wiederholter Closed Learning Loop ist noch nicht als Fakt belegt. | supported |
| C-005 | Designentscheidung | Controlled Selection soll erst nach staerkerem Closed Feedback kommen. | intentional |

## Leseregeln
- `supported` bedeutet: fuer eine gebundene Aussage derzeit tragfaehig.
- `partially-supported` bedeutet: plausibel, aber weiterhin gut angreifbar.
- `intentional` markiert Governance-Entscheidungen statt Beobachtungen.
- Auf private Quellen gestuetzte Claims sollten oeffentlich bis auf Weiteres als vorlaeufig gelesen werden.
