# Measurement Limits

## Why this page exists
A public evidence-first blueprint should disclose what its current evidence cannot yet prove.

## Current limits
### 1. Private-source dependence
The strongest current evidence is still internal/private.
That means public readers cannot yet independently inspect all raw materials.

### 2. Coverage limits
Coverage is not yet fully quantified across all task types, workflows, and failure modes.
Low coverage can look like improvement or regression when it is really sparse evidence.

### 3. Attribution limits
A future closed-loop proof must separate a real effect from background drift, instrumentation changes, or unrelated runtime changes.
That standard is not yet fully operationalized.

### 4. Existence vs repeated effectiveness
The presence of routing, review, or staging infrastructure does not prove that it works repeatedly as closed-loop infrastructure.

### 5. Public evidence gap
Publicly inspectable surrogates, excerpts, or redacted evidence packages are still missing.

## Practical reading rule
The right current public posture is:
- modest claims
- explicit caveats
- challengeable inference
- conservative governance language
