# System Scope

## Public definition
In this blueprint, **Hermes** refers to a learning-capable agent system composed of:
- runtime behavior
- measurement and evaluation surfaces
- signal routing and review paths
- persistence/inheritance mechanisms
- governance constraints around adaptation

## What is being classified
The phase model classifies the **system's learning-loop maturity**, not general intelligence, not raw model quality, and not product polish.

## Unit of analysis
Unless stated otherwise, phase assignment refers to the **whole learning loop of the current system configuration**.
It does **not** automatically generalize to:
- every deployment
- every workflow
- every subcomponent
- every future version

## What is in scope
- observability and fitness visibility
- signal generation and routing
- review/inbox/adoption paths
- runtime traceability after change
- measured effect and reversibility

## What is out of scope
- benchmark leadership claims
- autonomous self-modification as a goal
- broad AGI-style capability claims
- unsupervised optimization without audit

## Why this page exists
A skeptical reader should be able to answer one question quickly:

> What exactly is the thing this roadmap is talking about?
